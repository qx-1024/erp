# 进销存管理系统后端设计文档

## 1. 系统概述

### 1.1 系统背景

在当今竞争激烈的商业环境中，企业对于采购、销售和库存管理的效率要求越来越高。传统的手工管理方式已经难以满足企业快速发展的需求，因此开发一套高效、稳定、易用的进销存管理系统具有重要的现实意义。

本系统旨在为中小型企业提供一个全面的进销存解决方案，帮助企业实现采购、销售和库存管理的自动化和信息化，提高企业运营效率，降低运营成本。

### 1.2 系统目标

1.  实现采购、销售、库存等核心业务流程的自动化管理

2.  提供实时的业务数据查询和分析功能，为企业决策提供支持

3.  建立完善的用户权限管理体系，保障系统数据安全

4.  优化业务流程，提高工作效率，降低企业运营成本

5.  提供友好、易用的操作界面，降低用户学习成本

6.  系统具备良好的扩展性和可维护性，以适应企业业务发展的需求

### 1.3 系统范围

本系统主要涵盖企业进销存管理的核心业务流程，包括采购管理、销售管理、库存管理、财务管理、基础数据管理、用户权限管理等功能模块。系统将实现这些模块之间的数据共享和业务协同，形成一个完整的企业管理信息系统。

## 2. 系统架构设计

### 2.1 总体架构

本系统采用 Spring Cloud 微服务架构，将整个系统拆分为多个独立的微服务，每个微服务专注于特定的业务功能，并通过轻量级的通信机制进行协作。这种架构设计具有良好的扩展性、可维护性和高可用性，能够满足企业不断变化的业务需求。

系统总体架构分为以下几层：

1.  **客户端层**：包括 Web 客户端、移动端等各种前端应用，负责与用户进行交互。

2.  **网关层**：由网关模块组成，统一处理外部请求的路由、负载均衡、权限校验等，是系统对外的统一入口。

3.  **服务层**：由各个业务模块组成，包括基础数据模块、采购管理模块、销售管理模块、库存管理模块、财务管理模块、用户权限模块、系统配置模块、报表分析模块等，各模块之间通过 OpenFeign 进行服务调用。

4.  **公共层**：由公共模块组成，提供通用功能和资源，如实体类定义、工具类、通用配置、全局异常处理等，供各业务模块共享使用。

5.  **数据层**：包括 MySQL 数据库、Redis 缓存、RabbitMQ 消息队列等，提供数据存储、缓存和消息传递服务。

6.  **支撑服务**：包括注册中心、配置中心、监控中心、日志中心等，提供微服务运行所需的各种支撑服务。

### 2.2 微服务模块设计

根据业务功能和职责，将系统划分为以下微服务模块：

1.  **基础数据模块**：负责管理系统的基础数据，如商品信息、供应商信息、客户信息、员工信息、仓库信息等。

2.  **采购管理模块**：负责处理企业的采购业务流程，包括采购申请、采购订单、采购入库、采购退货等。

3.  **销售管理模块**：负责处理企业的销售业务流程，包括销售订单、销售出库、销售退货、客户信用管理等。

4.  **库存管理模块**：负责管理企业的库存信息，包括入库管理、出库管理、库存调拨、库存盘点、库存预警等。

5.  **财务管理模块**：负责处理企业的财务业务，包括应收账款管理、应付账款管理、费用管理、固定资产管理、总账管理、财务报表等。

6.  **用户权限模块**：负责管理系统的用户、角色和权限，实现用户身份认证和权限控制。

7.  **系统配置模块**：负责管理系统的各种配置信息，如系统参数、菜单管理、数据字典、消息模板等。

8.  **报表分析模块**：负责生成各种业务报表和进行数据分析，为企业决策提供支持。

9.  **公共模块**：提供通用功能和资源，供各业务模块共享使用。

10. **网关模块**：统一处理外部请求的路由、负载均衡、权限校验等。

11. **认证授权模块**：负责用户身份认证和权限授权。

### 2.3 技术选型

#### 2.3.1 后端技术栈

1.  **Spring Cloud**：微服务框架，提供服务发现、配置管理、负载均衡、断路器等功能。
2.  **Spring Boot**：简化 Spring 应用开发，快速搭建微服务。
3.  **MyBatis-Plus**：ORM 框架，简化数据库操作。
4.  **MySQL**：关系型数据库，存储系统业务数据。
5.  **Redis**：缓存数据库，提高系统性能。
6.  **RabbitMQ**：消息队列，实现系统间的异步通信和解耦。
7.  **Spring Security**：安全框架，实现用户认证和权限管理。
8.  **Nacos**：服务发现和配置中心，提供微服务的注册、发现和配置管理。
9.  **Sentinel**：流量控制和熔断降级框架，保障系统稳定性。
10.  **Gateway**：API 网关，统一处理请求路由、负载均衡、权限校验等。
11.  **OpenFeign**：声明式服务调用框架，简化微服务间的调用。
12.  **Seata**：分布式事务解决方案，保证跨服务调用的数据一致性。
13.  **ELK**：日志收集、存储和分析系统，方便系统监控和问题排查。
14.  **Docker**：容器化技术，实现应用的快速部署和资源隔离。
15.  **Jenkins**：持续集成和持续部署工具，实现自动化构建和部署。
16. **Maven**：项目构建和依赖管理工具。
17.  **Activiti**：工作流审批。

#### 2.3.2 前端技术栈

1.  **Vue.js**：前端框架，构建用户界面。

2.  **Element UI**：UI 组件库，提供美观、易用的界面组件。

3.  **Axios**：HTTP 客户端，用于与后端 API 进行通信。

4.  **Vue Router**：路由管理，实现单页面应用的路由功能。

5.  **Vuex**：状态管理，管理应用的全局状态。

6.  **Vue CLI**：项目脚手架，快速搭建前端项目。

7.  **Tailwind CSS**：CSS 工具类框架，快速构建美观的界面。

8.  **Font Awesome**：图标库，提供丰富的图标资源。

### 2.4 数据存储设计

#### 2.4.1 数据库设计原则

1.  遵循第三范式，减少数据冗余

2.  合理设计表结构，确保数据完整性和一致性

3.  为高频查询字段添加索引，提高查询效率

4.  考虑数据库性能和扩展性，避免单表数据量过大

5.  设计合理的数据库备份和恢复策略，保障数据安全

#### 2.4.2 数据库表结构设计

以下是系统主要模块的表结构设计概述：

-   **基础数据表**：

    -   商品表：存储商品基本信息，如商品 ID、名称、规格、型号、分类、单位、成本价、销售价等

    -   供应商表：存储供应商信息，如供应商 ID、名称、联系人、电话、地址、银行账户等

    -   客户表：存储客户信息，如客户 ID、名称、联系人、电话、地址、信用额度等

    -   员工表：存储员工信息，如员工 ID、姓名、部门、职位、联系方式等

    -   仓库表：存储仓库信息，如仓库 ID、名称、地址、负责人等

-   **采购管理表**：

    -   采购申请表：存储采购申请信息，如申请 ID、申请日期、申请人、申请原因、状态等

    -   采购订单表：存储采购订单信息，如订单 ID、供应商 ID、订单日期、订单金额、状态等

    -   采购入库表：存储采购入库信息，如入库 ID、订单 ID、仓库 ID、入库日期、入库金额等

    -   采购退货表：存储采购退货信息，如退货 ID、订单 ID、仓库 ID、退货日期、退货金额等

-   **销售管理表**：

    -   销售订单表：存储销售订单信息，如订单 ID、客户 ID、订单日期、订单金额、状态等

    -   销售出库表：存储销售出库信息，如出库 ID、订单 ID、仓库 ID、出库日期、出库金额等

    -   销售退货表：存储销售退货信息，如退货 ID、订单 ID、仓库 ID、退货日期、退货金额等

    -   客户信用表：存储客户信用信息，如客户 ID、信用额度、已使用额度、信用期限等

-   **库存管理表**：

    -   库存表：存储商品库存信息，如库存 ID、商品 ID、仓库 ID、库存数量、库存成本等

    -   入库明细表：存储入库明细信息，如明细 ID、入库 ID、商品 ID、数量、单价等

    -   出库明细表：存储出库明细信息，如明细 ID、出库 ID、商品 ID、数量、单价等

    -   库存流水表：记录库存变动历史，如流水 ID、商品 ID、仓库 ID、变动类型、变动数量、变动时间等

    -   库存盘点表：存储库存盘点信息，如盘点 ID、仓库 ID、盘点日期、盘点人、状态等

-   **财务管理表**：

    -   应收款表：存储应收账款信息，如应收款 ID、客户 ID、订单 ID、金额、到期日、状态等

    -   应付款表：存储应付账款信息，如应付款 ID、供应商 ID、订单 ID、金额、到期日、状态等

    -   费用表：存储费用信息，如费用 ID、费用类型、金额、日期、申请人、审批人等

    -   固定资产表：存储固定资产信息，如资产 ID、名称、型号、购置日期、原值、折旧方法等

    -   总账表：存储总账信息，如凭证 ID、日期、摘要、借方金额、贷方金额、科目等

-   **用户权限表**：

    -   用户表：存储系统用户信息，如用户 ID、用户名、密码、姓名、邮箱、状态等

    -   角色表：存储系统角色信息，如角色 ID、角色名称、角色描述等

    -   权限表：存储系统权限信息，如权限 ID、权限名称、权限代码、权限类型等

    -   用户角色关联表：关联用户和角色

    -   角色权限关联表：关联角色和权限

-   **系统配置表**：

    -   系统参数表：存储系统参数信息，如参数 ID、参数名称、参数值、参数描述等

    -   菜单表：存储系统菜单信息，如菜单 ID、菜单名称、菜单 URL、父菜单 ID、排序号等

    -   数据字典表：存储系统数据字典信息，如字典 ID、字典类型、字典代码、字典值等

    -   消息模板表：存储系统消息模板信息，如模板 ID、模板类型、模板内容、备注等

### 2.5 服务间通信设计

系统采用以下方式实现微服务之间的通信：

1.  **RESTful API**：各微服务通过 RESTful API 提供服务接口，使用 JSON 格式进行数据交换。

2.  **OpenFeign**：服务间调用使用 OpenFeign 实现声明式调用，简化服务调用代码。

3.  **消息队列**：使用 RabbitMQ 实现异步通信和解耦，提高系统性能和可靠性。

4.  **服务注册与发现**：使用 Nacos 实现服务的注册与发现，使服务间调用无需硬编码服务地址。

### 3.2 采购管理模块

#### 3.2.1 功能概述

采购管理模块负责处理企业的采购业务流程，包括采购申请、采购订单、采购入库、采购退货等。该模块实现了采购业务的全程跟踪和管理，提高采购效率，降低采购成本。

#### 3.2.2 功能详细设计

-   **采购申请管理**

    -   功能：处理采购申请的创建、审批、查询等

    -   接口设计：


    创建采购申请：接收采购申请信息，包括申请商品、数量、申请原因等，验证数据有效性后插入数据库
    
    提交采购申请：将采购申请提交审批，更新申请状态
    
    审批采购申请：根据审批结果更新采购申请状态
    
    查询采购申请：支持按条件查询采购申请列表和采购申请详情
    
    导出采购申请：将采购申请信息导出为 Excel 或 PDF 格式

-   **采购订单管理**

    -   功能：管理采购订单的创建、审核、下发、跟踪等

    -   接口设计：


    创建采购订单：根据采购申请或直接创建采购订单，接收供应商信息、商品信息、价格、数量等，验证数据有效性后插入数据库
    
    审核采购订单：对采购订单进行审核，更新订单状态
    
    下发采购订单：将审核通过的采购订单下发给供应商
    
    查询采购订单：支持按条件查询采购订单列表和采购订单详情
    
    跟踪采购订单：查看采购订单的执行状态和物流信息
    
    修改采购订单：在订单未执行前可修改订单信息
    
    取消采购订单：取消未执行的采购订单

-   **采购入库管理**

    -   功能：处理采购入库业务，包括入库单的创建、审核、入库确认等

    -   接口设计：


    创建采购入库单：根据采购订单创建入库单，接收实际入库商品信息、数量等，验证数据有效性后插入数据库
    
    审核采购入库单：对入库单进行审核，更新入库单状态
    
    确认采购入库：将审核通过的入库单进行入库确认，更新库存信息
    
    查询采购入库单：支持按条件查询采购入库单列表和采购入库单详情
    
    生成入库报表：生成采购入库统计报表

-   **采购退货管理**

    -   功能：处理采购退货业务，包括退货单的创建、审核、退货出库等

    -   接口设计：


    创建采购退货单：根据采购入库单创建退货单，接收退货商品信息、数量、原因等，验证数据有效性后插入数据库
    
    审核采购退货单：对退货单进行审核，更新退货单状态
    
    执行采购退货：将审核通过的退货单进行退货出库操作，更新库存信息
    
    查询采购退货单：支持按条件查询采购退货单列表和采购退货单详情

## 5. 系统安全设计

### 5.1 身份认证设计

系统采用基于 JWT（JSON Web Token）的身份认证机制，主要流程如下：

1.  用户登录时，提交用户名和密码到认证接口

2.  认证接口验证用户名和密码的有效性

3.  如果验证通过，生成 JWT 令牌并返回给客户端

4.  客户端在后续请求中携带 JWT 令牌

5.  服务端验证 JWT 令牌的有效性，获取用户信息

6.  根据用户信息进行权限校验

### 5.2 权限控制设计

系统采用基于 RBAC（Role-Based Access Control）的权限控制模型，主要设计如下：

1.  系统定义多种角色，如管理员、采购专员、销售专员、库存管理员等

2.  每个角色关联一组权限，权限可以是菜单访问权限、按钮操作权限、数据访问权限等

3.  用户分配一个或多个角色

4.  系统在用户访问资源时，根据用户的角色和权限进行校验

5.  支持数据权限控制，如用户只能查看和操作自己负责的业务数据

### 5.3 数据加密设计

系统对敏感数据进行加密处理，主要设计如下：

1.  用户密码采用哈希算法加密存储，使用盐值增强安全性

2.  数据库中的敏感数据（如银行卡号、身份证号等）采用对称加密算法加密存储

3.  系统与第三方系统之间的通信数据采用 SSL/TLS 加密

4.  重要业务数据在传输过程中采用 HTTPS 协议进行加密

### 5.4 安全审计设计

系统提供安全审计功能，主要设计如下：

1.  记录用户的登录行为，包括登录时间、登录 IP、登录状态等

2.  记录用户的操作行为，包括操作时间、操作内容、操作人等

3.  记录系统的异常事件，包括异常发生时间、异常类型、异常详情等

4.  提供安全审计报表，方便管理员查看和分析安全事件

5.  设置安全审计规则，对异常行为进行实时监控和预警
